[Title] [Bug][Numpy] Wrong gradient of np.where

## Description
Example 1: Using np.where(array, array, scalar)

```python
import mxnet as mx
mx.npx.set_np()

a = mx.np.array([1, 0, 1])
b = mx.np.array([2, 3, 4])

b.attach_grad()

with mx.autograd.record():
    c = mx.np.where(a, b, -1)
    c.backward()
print(b.grad)
```
Output: [0. 1. 0.]

Example 2: Using np.where(array, array, array)
```python

import mxnet as mx
mx.npx.set_np()

a = mx.np.array([1, 0, 1])
b = mx.np.array([2, 3, 4])

b.attach_grad()

with mx.autograd.record():
    c = mx.np.where(a, b, mx.np.array([-1, -1, -1]))
    c.backward()
print(b.grad)
```
Output: [1. 0. 1.]

The second one is correct.
