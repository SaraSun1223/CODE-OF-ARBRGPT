[Title] [ONNX] Invalid ONNX graph when exporting prim__Constant of empty tensor data

## ðŸ› Bug

torch.onnx exports invalid ONNX for scalar `torch.zeros` followed by an inplace op.

## To Reproduce

Steps to reproduce the behavior:

1. Run the following script

```
import torch


class Zeros(torch.nn.Module):
    def __init__(self):
        super().__init__()

    def forward(self, x):
        y = torch.zeros(())
        y += x
        return y


x = torch.tensor(42.)
torch.onnx.export(Zeros(), x, 'zeros.onnx')
```

You will see 

```
==> Context: Bad node spec: output: "1" name: "Constant_0" op_type: "Constant" attribute { name: "value" type: TENSORS }
```

If you show the ONNX by disabling onnx checker and shape inference (ONNX's shape inference SEGVs for the invalid ONNX in question) by

```
torch.onnx._export(
    Zeros(), x, 'zeros.onnx',
    enable_onnx_checker=False,
    onnx_shape_inference=False,
)
print(onnx.load('zeros.onnx'))
```

you will see

```
  node {
    output: "1"
    name: "Constant_0"
    op_type: "Constant"
    attribute {
      name: "value"
      type: TENSORS
    }
  }
```

so it seems the parameter for `torch.zeros`, which is `()`, is translated to `TENSORS` not a scalar `TENSOR`.


## Expected behavior

This script does not raise an error. Note torch 1.7.1 did not have this issue and it seems this issue is introduced in 1.8.0.

## Additional context

There seem to be a few workarounds like replacing `y += x` by `y = y + x` or using `torch.scalar_tensor(0.)` instead of `torch.zeros(())`.
