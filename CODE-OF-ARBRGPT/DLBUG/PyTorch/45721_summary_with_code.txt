[Title] kthvalue incorrect with strided GPU tensor

## ðŸ› Bug

When input tensor is on GPU and is strided, `kthvalue` will give wrong values and indices.

## To Reproduce

```
>>> import torch
>>> a = torch.tensor([0, -1, 1, -2, 2]).cuda()[::2]
>>> a
tensor([0, 1, 2], device='cuda:0')
>>> a.kthvalue(2)
torch.return_types.kthvalue(
values=tensor(0, device='cuda:0'),
indices=tensor(0, device='cuda:0'))
```

## Expected behavior

The behavior on CPU as below is expected:
```
>>> import torch
>>> a = torch.tensor([0, -1, 1, -2, 2])[::2]
>>> a
tensor([0, 1, 2])
>>> a.kthvalue(2)
torch.return_types.kthvalue(
values=tensor(1),
indices=tensor(1))
```
It seems that `kthvalue` is performed on `[0, -1, 1]`, the tensor with the same start address and size as `a` but stride 1.
A workaround is to use `a.contiguous().kthvalue(2)`, but this is not the expected way to use `kthvalue`.
